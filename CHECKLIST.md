# ✅ Чек-лист проверки приложения «Код Мастер»

## 1. Запуск и главное окно
- [x] Приложение запускается на Python 3.12+ без ошибок импорта
- [x] Главное окно фиксированного размера 480×320
- [x] Заголовок окна: «Код Мастер»
- [x] Фон окна: #2B2B2B, текст белый #FFFFFF
- [x] Шрифт Segoe UI 10pt (на Windows) / системный sans-serif (на Mac)
- [x] Кнопки «Обновить» и «Настроить» в правом верхнем углу
- [x] Размер кнопок 130×34px, скругление 17px, чёрный фон #1A1A1A
- [x] Анимация наведения (#333333) и нажатия (#4D4D4D) плавная (200ms)
- [x] ProgressBar под кнопками, скрыт по умолчанию
- [x] Статусная строка «Готов к работе» внизу
- [x] Индикатор порта (зелёный/красный/серый кружок) + название порта
- [x] Кнопка «📄 Логи» открывает папку с лог-файлом

## 2. Модальное окно выбора COM-порта
- [x] Открывается при первом нажатии функциональной кнопки, если порт не выбран
- [x] Выпадающий список доступных портов (реальные + эмулированный, если включён)
- [x] Выбор скорости (список, по умолчанию 115200)
- [x] Чекбокс «Режим эмуляции» (скрытый или малозаметный)
- [x] Кнопка «Подключить»: успех – закрыть, ошибка – сообщение
- [x] Закрытие диалога без выбора = отмена, порт не сохранён

## 3. Окно «Настройка» (вкладки)
- [x] Открывается немодально, размер 900×650
- [x] Вкладки: «CAN Тригер», «Мониторинг CAN», «CAN Шлюз»
- [x] Кнопки «Сохранить конфигурацию в файл» и «Загрузить конфигурацию из файла» работают

### 3.1 Вкладка «CAN Тригер»
- [x] 10 блоков-триггеров с полями ID, D0…D7 (HEX) и чекбоксом «Активен»
- [x] Для каждого: поля ответа ID отв. и D0…D7 отв.
- [x] Кнопка «Тест» отправляет тестовый пакет
- [x] Кнопка «Применить триггеры» запускает проверку триггеров
- [x] При совпадении с активным триггером отправляется ответ в тот же канал
- [x] Кнопка «Остановить» останавливает обработку триггеров

### 3.2 Вкладка «Мониторинг CAN»
- [x] Разделена вертикально на «CAN 1» и «CAN 2» (QSplitter)
- [x] Для каждого канала: кнопки «Старт», «Стоп», «Очистить»
- [x] Поле фильтра ID (пустое – без фильтра)
- [x] Чекбокс «Приостановить» (заморозка таблицы)
- [x] Таблица: время (HH:MM:SS.fff), ID (HEX), D0–D7 (HEX), DLC
- [x] Тёмный фон ячеек #1E1E1E, сетка #333333, лимит 50 000 строк
- [x] Панель ручной отправки: ID + 8 Data, кнопка «Отправить»
- [x] Отправленный вручную пакет подсвечивается цветом #4A6A8A
- [x] Статистика: «Принято: N | Скорость: N пак/с»
- [x] Глобальные кнопки «Запустить оба», «Остановить оба», «Очистить всё»

### 3.3 Вкладка «CAN Шлюз»
- [x] Секция «Игнорировать»: 10 полей ID (две колонки), чекбокс «Включить игнорирование»
- [x] Секция «Правила подмены»: 10 блоков с приёмом/подменой ID+Data и направлением
- [x] Кнопки «Запустить шлюз» и «Остановить»
- [x] Логика: ретрансляция, игнорирование, подмена между каналами

## 4. Функция «Обновить» (прошивка)
- [x] Открывается диалог выбора .bin/.hex
- [x] Запускается фоновый поток
- [x] Реализован протокол STM32 UART bootloader (AN3155)
- [x] Настройка порта (Even, 1 стоп-бит), управление DTR/RTS
- [x] Синхронизация 0x7F, ожидание ACK, повторы при NACK
- [x] Стирание (0x43/0x44) с обработкой ответов
- [x] Запись блоками по 256 байт (0x31), контроль XOR, повторы при ошибках
- [x] Прогресс-бар обновляется
- [x] Сообщение об успехе/ошибке после завершения

## 5. Протокол CAN-кадров (can_protocol.py)
- [x] Упаковка (pack_can_frame) с маркером 0xBB, байтом канала, 2-байтным ID (LE), длиной, данными, XOR-суммой
- [x] Распаковка (unpack_can_frame) с поиском маркера 0xAA, проверкой суммы
- [x] Возвращает словарь {channel, id, data} или None при ошибке

## 6. Многопоточность и SerialManager
- [x] Открытие/закрытие порта в отдельном QThread
- [x] Чтение в бесконечном цикле, буферизация, парсинг кадров
- [x] Сигналы new_can_frame и error_occurred подключены к UI
- [x] Отправка данных через потокобезопасный метод с lock
- [x] Корректное завершение потока при закрытии порта и приложения

## 7. Настройки и логирование
- [x] Класс Config сохраняет/загружает настройки в config.json
- [x] Экспорт/импорт профилей (триггеры, шлюз, игнорирование) через файлы
- [x] Логирование в файл с ротацией (1 МБ, 3 файла)
- [x] Логируются все CAN-пакеты, триггеры, ошибки, этапы прошивки

## 8. Тёмная тема и кроссплатформенность
- [x] Единая функция apply_dark_theme() задаёт палитру и стили
- [x] Интерфейс корректно отображается на macOS и Windows
- [x] Эмулятор COM-порта (FakeSerial) работает на macOS для тестов
